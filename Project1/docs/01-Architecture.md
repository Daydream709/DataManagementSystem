# 1. 项目整体架构说明

## 1.1 架构目标

本系统采用前后端分离架构，确保：

1. 业务逻辑集中在后端，便于迭代第二阶段需求。
2. 前端通过 REST API 解耦，便于替换 UI 与组件。
3. MongoDB 数据结构灵活，支持题目/跳转规则扩展。

## 1.2 技术栈

- 前端：Vue3 + Vite + TailwindCSS + Pinia + Vue Router + Axios
- 后端：Django + DRF + PyMongo（不使用关系型 ORM 建模业务）
- 鉴权：JWT（Bearer Token）
- 数据库：MongoDB

## 1.3 模块划分

### 前端模块

- 认证模块：登录/注册、Token 存储
- 问卷管理模块：创建问卷、发布状态切换、编辑入口
- 题目模块：单选/多选/填空题维护、校验规则配置
- 跳转规则模块：条件规则可视化配置
- 填写模块：按当前题目回答并动态跳题
- 统计模块：整卷统计与单题统计展示

### 后端模块

- 用户模块：注册、登录、JWT 签发与鉴权
- 问卷模块：问卷 CRUD、发布/关闭、链接生成
- 题目模块：题目 CRUD 与参数校验
- 跳转模块：规则 CRUD 与规则执行器
- 提交模块：路径计算、答案合法性校验、提交持久化
- 统计模块：单选/多选/填空统计计算

## 1.4 请求流

1. 用户登录后获得 JWT。
2. 前端 Axios 拦截器自动附带 Bearer Token。
3. DRF 自定义认证中间件解析 JWT 并验证用户。
4. 业务服务层访问 MongoDB 集合执行业务。
5. 返回统一响应结构：`{ code, message, data }`。

## 1.5 扩展性设计（面向第二阶段）

- 题型扩展：题目 `type + validation` 的 schema 可追加。
- 跳转扩展：规则使用 `operator + value + priority`，可增操作符。
- 统计扩展：统计逻辑按题型拆分，可新增题型聚合器。
- 权限扩展：已按 owner_id 隔离问卷，可扩展协作者角色。
- 提交流程扩展：提交路径引擎与校验引擎独立封装，便于插拔。
